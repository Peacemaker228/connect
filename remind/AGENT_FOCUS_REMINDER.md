# Agent Focus Reminder

## Purpose

Эту памятку можно отправлять перед сложной или длинной задачей, чтобы агент не терял фокус, не расползался по scope и не отвечал по инерции прошлого контекста.

## Universal Prompt

```text
Ответь только на мой вопрос.
Сначала сверись с кодом и актуальными доками.
Не делай предположений, если не проверил.
Не уходи в соседние темы.
В начале коротко повтори, как ты понял задачу.
```

## Stronger Version

```text
Работай строго в рамках текущего вопроса.
Сначала коротко повтори, как ты понял задачу.
Сверься с кодом и source of truth документами перед выводом.
Не придумывай, если не проверил.
Не расширяй scope без явного запроса.
Если ответ узкий, не уходи в roadmap, архитектуру или следующие этапы.
Если есть неопределенность, прямо скажи, что нужно проверить.
```

## Good Control Phrases

- `Ответь только про этот вопрос.`
- `Не переходи к следующему шагу.`
- `Сначала проверь код, потом отвечай.`
- `Сверься с актуальными доками и не делай предположений.`
- `Не уходи в соседние темы.`
- `Сначала скажи, какой сейчас active stage/wave.`
- `Сейчас только анализ.`
- `Сейчас только ревью.`
- `Сейчас только ответ на вопрос, без брифа.`

## When To Use

- когда задача длинная и много старого контекста
- когда вопрос узкий, а агент может поехать в соседнюю тему
- когда важна точность по текущему `stage/wave`
- когда нельзя допускать scope creep
- когда нужен ответ только по фактам из кода и docs

## Best Practice

Лучший способ держать агента в порядке:

1. Явно ограничивать scope вопроса.
2. Якорить на source of truth.
3. Просить короткий re-state задачи в начале.
4. Разделять режимы: `анализ`, `ревью`, `следующий шаг`, `только ответ`.
5. Сразу останавливать, если агент уходит не туда.

## Short Reset Template

```text
Сначала повтори, как ты понял задачу.
Потом сверяйся только с кодом и актуальными доками.
Не придумывай, если не проверил.
Не уходи за пределы этого вопроса.
```
