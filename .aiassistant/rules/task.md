Please solve this as a senior frontend developer.

Before changing code:

inspect the relevant files and nearby modules
understand the existing architecture and patterns
identify whether the current implementation already follows a project convention
When proposing the solution:

prefer the most correct and minimally invasive option
keep the code in the spirit of the current project
if the current pattern is weak or risky, explain that and suggest a better alternative
think about runtime behavior, not only TypeScript types
At the end, always provide:

short analysis of the issue
the chosen solution
changed files
what changed in each file
any risks or follow-up notes
