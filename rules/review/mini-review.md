Проведи ревью как senior/lead engineer.

Не опирайся только на сводку агента. Самостоятельно проверь:
- git status / branch / последние коммиты
- diff и затронутые файлы
- соответствие задаче, brief/docs и текущему плану
- runtime-риски, fallback/compatibility, edge cases
- оправданность изменений в документации
- нужны ли тесты или проверки

Дай вывод в формате:
1. Блокирующие замечания, если есть
2. Некритичные риски/замечания
3. Что проверено командами
4. Можно ли коммитить/мержить
5. Что следующий логичный шаг

Если видишь проблему — объясни конкретно: файл, строка/место, почему это риск, как лучше исправить.
Не придумывай план из памяти: сначала сверяйся с кодом и актуальными docs.
